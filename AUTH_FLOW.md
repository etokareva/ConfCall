# Авторизация и приглашения в группы

## Перед началом

Приложение должно быть запущено:

```bash
npm run db:up
npm run db:push
npm run dev:be
npm run dev:fe
```

Frontend открывается по адресу:

```text
http://127.0.0.1:4200/
```

## 1. Войти под администратором

1. Откройте страницу входа:

   ```text
   http://127.0.0.1:4200/login
   ```

2. Нажмите `Dev-вход администратора`.

3. После входа откройте `Группы`.

Это тестовый администратор для локальной разработки. Он создается автоматически при первом dev-входе.

## 2. Создать группу и отправить приглашения

1. В верхнем меню откройте `Настройки`.

2. В блоке `Группы и доступ участников` создайте группу, например:

   ```text
   Программный комитет 2026
   ```

3. Выберите созданную группу.

4. Введите список email через запятую или с новой строки.

5. Подтвердите список в модальном окне и нажмите `Отправить приглашения`.

6. В списке приглашений появятся адреса и статус отправки.

Каждый приглашённый получит письмо со ссылкой на `/invite?token=...`.

## 3. Как пользователь присоединяется к группе

1. Пользователь открывает ссылку из письма.

2. Если он уже вошёл в приложение и email совпадает, он может сразу нажать `Присоединиться к группе`.

3. Если аккаунта ещё нет, открывается форма регистрации по ссылке приглашения.

4. После регистрации или принятия приглашения пользователь автоматически попадает в группу.

## 4. Проверить ограничения

- Ссылка приглашения одноразовая и имеет срок действия.
- Приглашение можно принять только с email, указанного в письме.
- Если email ещё не зарегистрирован, регистрация возможна только по invite-ссылке.
- На `/login` остаются только вход по email и паролю, dev-вход и сброс пароля.

## 5. Вернуться под администратором

1. Нажмите `Выйти`.

2. Откройте:

   ```text
   http://127.0.0.1:4200/login
   ```

3. Нажмите `Dev-вход администратора`.

Администратор снова сможет управлять своими группами и отправлять приглашения.
