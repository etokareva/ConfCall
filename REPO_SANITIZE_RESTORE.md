# Подготовка проекта к открытому репозиторию

В проекте реальные чувствительные данные вынесены в локальный файл `.env.restore.local`.

Этот файл уже игнорируется через шаблон `.env.*.local` в `.gitignore`, поэтому он не должен попасть в публичный репозиторий, если не добавлять его вручную.

## Что очищено

- `.env` заменён на безопасные заглушки.
- `docker-compose.yml` переведён на переменные окружения вместо жёстко зашитых MySQL-паролей.
- Документация очищена от реальных email, паролей и токенов.

## Как подготовить проект к публикации

1. Убедиться, что `.env.restore.local` лежит только локально и не попал в архив или commit.
2. При необходимости временно убрать `.env.restore.local` из папки проекта.
3. Публиковать проект с текущими заглушками в `.env`, `.env.example` и документации.

## Как потом восстановить рабочую конфигурацию

1. Вернуть файл `.env.restore.local` в корень проекта.
2. Скопировать значения из `.env.restore.local` обратно в `.env`.
3. При необходимости перезапустить backend и MySQL.

## Какие значения лежат в локальном файле

- MySQL database, user, password, root password
- `DATABASE_URL`
- `TELEMOST_OAUTH_TOKEN`
- SMTP user, password и `MAIL_FROM`
- `APP_PUBLIC_URL`
